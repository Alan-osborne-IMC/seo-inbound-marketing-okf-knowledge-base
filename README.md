# SEO Inbound Marketing OKF Knowledge Base
This repository contains an OKF-structured knowledge base for Alan Osborne and seo-inbound-marketing.com. It is designed to make SEO, inbound marketing, local SEO, semantic SEO, and AI-search knowledge easier for LLMs, agents, and RAG systems to read, retrieve, and use.

Purpose
The purpose of this repository is to organize business and marketing knowledge into a format that is both human-readable and machine-friendly. It supports AI retrieval, research workflows, content strategy, service documentation, and knowledge reuse across SEO, inbound marketing, and answer engine optimization use cases.

What this repository includes
This repository may include:

OKF-flattened marketing and business knowledge files.

SEO strategy notes, frameworks, checklists, and implementation guidance.

Inbound marketing concepts, content systems, and supporting documentation.

Service documentation, deliverables, and positioning materials.

Local SEO, semantic SEO, and AI-search optimization resources.

Structured markdown files intended for LLM and RAG ingestion workflows.

Suggested folder structure
services/ — service-related assets and documentation.

services/seo-compass-audit/ — flagship audit framework, positioning, deliverables, and supporting files.

services/business-knowledge-optimization-okf/ — service materials for transforming business knowledge into structured, agent-readable OKF bundles.

services/local-seo/ — local SEO service playbooks, checklists, deliverables, and location-specific optimization assets.

service-pages/ — page copy and supporting content for service offers.

semantic-seo/ — entity SEO, topical authority, structured content, and knowledge modeling resources.

ai-search/ — answer engine optimization, LLM visibility, retrieval, and AI-search-related materials.

reference/ — notes, prompts, templates, glossaries, and reusable supporting documents.

Branding and ownership
This knowledge base is associated with Alan Osborne and the SEO Inbound Marketing brand at seo-inbound-marketing.com. Consistent use of the same brand name, domain, service names, and topic language across the repository, README, and linked web properties helps both people and machine systems understand the entity behind the content.

How this repository can be used
This repository is intended to support:

LLM and RAG ingestion workflows.

Internal knowledge organization and retrieval.

Service documentation and offer development.

Content planning and semantic content modeling.

AI-readable business knowledge packaging using OKF patterns.

Naming conventions
Recommended naming patterns:

Use lowercase file and folder names.

Separate words with hyphens.

Keep names descriptive but not overly long.

Include topic keywords where they improve clarity.

Avoid unnecessary abbreviations unless they are used consistently.

Examples:

local-seo-ranking-signals.md

semantic-seo-entity-mapping.md

inbound-content-offer-framework.md

business-knowledge-optimization-okf-overview.md

seo-compass-audit-deliverables.md

Usage notes
If this repository is used as a public AI-readable knowledge source, keep file names, folder names, headings, and markdown structure consistent. Clear titles, descriptive headings, and logical folder organization make the content easier for both collaborators and machine retrieval systems to parse and understand.

Maintenance
Update this repository as services evolve, source materials are flattened into OKF-compatible files, and new knowledge assets are added. For larger sections, a folder-level README.md can be added inside each service folder to describe scope, deliverables, and related files.
